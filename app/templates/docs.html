工作台 作品广场

API 与使用说明

浅夜の梦支持 GPT Image 2、Gemini / Grok 图像模型、参考图编辑、作品广场上传与评论。 图片 API 的 size 必须传像素值，例如 1024x1024，不要把 1:1 这类比例直接传给 API。

网站使用流程

  1. 进入 /play，选择 API 线路，输入 API Key，点击「连接」。
  2. 连接成功后选择模型。支持参考图的模型会显示参考图上传区。
  3. 填写提示词，选择分辨率、比例、质量、输出格式等参数。
  4. 点击「生成图片」。生成结果会显示在结果画布中，并保存在本地历史；已登录用户可启用服务器历史。
  5. 结果卡片可预览、下载、查看提示词、删除；开启作品广场后可上传公开作品。
  6. 作品广场的浏览量来自真实打开行为：打开作品弹窗或详情页时才会 +1。

API 线路

工作台下拉框会真正切换请求目标；后续模型列表、生成、任务轮询都会使用当前选中的线路。

名称Base URL
主线路https://api.example.com

认证与请求模式

所有上游 API 请求都使用 Authorization: Bearer YOUR_API_KEY

curl https://api.example.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"

本站工作台支持两种内部请求模式：frontend 代表浏览器直连所选 API 线路； backend 代表浏览器请求本站 /api/proxy，由本站服务端转发到当前线路。 外部开发者通常直接使用上面的 Base URL。

文生图

POST /v1/images/generations

{
  "model": "gpt-image-2",
  "prompt": "A quiet anime-style girl reading beside a rainy window",
  "n": 1,
  "size": "1536x864",
  "quality": "auto",
  "background": "auto",
  "moderation": "auto",
  "output_format": "png"
}

当前兼容接口中，gpt-image-2 文生图与图生图都以 size 为准，不再依赖 resolution。 quality 可选 auto、low、medium、high。 output_format 可选 png、jpeg、webp。 当格式是 jpeg 或 webp 时，可额外传 output_compression，范围 0-100。

图生图 / 参考图编辑

POST /v1/images/edits multipart/form-data

curl -X POST "https://api.example.com/v1/images/edits" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=保持人物姿势，把画面改成黑白漫画风格" \
  -F "image=@reference.png" \
  -F "n=1" \
  -F "size=1024x1024" \
  -F "quality=high" \
  -F "output_format=png"

多张参考图时重复传 image 字段。工作台最多保留 4 张参考图，并会在浏览器端压缩到适合上传的大小；当前兼容接口中图生图以 size 为准，不再依赖 resolution。

Chat 图像模型

POST /v1/chat/completions

{
  "model": "gemini-3.1-flash-image",
  "stream": false,
  "messages": [
    {
      "role": "user",
      "content": [
        { "type": "text", "text": "Make the aspect ratio 16:9, 画一座雨夜城市" }
      ]
    }
  ]
}

Gemini 支持图文数组 content，可附加 image_url 参考图。 Grok 配置为纯字符串 content，且不自动注入比例描述。

异步任务与轮询

如果上游返回 task_id，工作台会自动轮询 /v1/tasks/{task_id} 直到拿到图片或超时。

GET /v1/tasks/img_xxxxxxxx
Authorization: Bearer YOUR_API_KEY

{
  "task_id": "img_xxxxxxxx",
  "status": "success",
  "data": [
    { "url": "https://..." }
  ]
}

响应格式

工作台会兼容 base64、URL 和 task_id。推荐优先读取 data 数组。

{
  "created": 1770000000,
  "data": [
    { "b64_json": "iVBORw0KGgo..." }
  ]
}
{
  "created": 1770000000,
  "data": [
    { "url": "https://example.com/image.png" }
  ]
}

常见错误

现象处理方式
模型列表为空检查 API Key、线路是否可用，以及该 Key 是否有模型权限。
size 不生效确认请求中传的是 1024x1024 这类像素值，不是 1:1。
参考图失败检查文件格式，建议 PNG / JPEG / WebP，单图不要过大。
任务超时高质量或 4K 生成耗时更长，可降低分辨率或稍后重试。
作品无法上传需要登录，并且不能重复上传相同图片。
